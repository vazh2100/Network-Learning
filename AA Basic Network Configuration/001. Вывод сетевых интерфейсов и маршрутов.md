`ifconfig` - interface config - устаревшая команда для настройки сетевых интерфейсов

```bash
ip address    # Отображение информацию об ip адресах сетевых интерфесов
или
ifconfig
```

```bash
ip -4 address # Отображение только ip адресов версии 4
```

`inet 127.0.0.1/8` - ip адрес интерфейса

```bash
ip route # Вывод сведений о маршрутизации
или
netstat -rn
или
route
```

`default via 10.42.41.131 dev wlan0 proto dhcp src 10.42.41.1 metric 600`
Если адрес назначения пакета не входит в диапазон более частной записи о маршруте, то есть его нет в таблице
маршрутизации,
то он будет обработан записью по умолчанию default(0.0.0.0/0).
`via 10.42.41.131` - означает, что в случае, если запись по умолчанию выбрана, то такой пакет отправляется на шлюз
`10.42.41.131`(например, роутер). Предполагается, что шлюз(роутер), согласно своей таблице маршрутизации, знает куда
дальше
направить пакет.
`10.42.41.1` - это адрес wlan0 интерфейса на текущем хосте.

`10.42.41.0/24 dev wlan0 proto kernel scope link src 10.42.41.1 metric 600 `
`10.42.41.0/24` - это локальная подсеть. Эта запись говорит о том, что это устройство находится в этой локальной
подсети.
С другой стороны, это говорит о том, что если адрес назначения пакета входит в эту подсеть, то пакет направится через
интерфейс
wlan0 по второму уровню osi, то есть без ip маршрутизации на роутере. Роутер выступит в роли коммутатора, а не
маршрутизатора.

`169.254.0.0` - подсеть локальных адресов связи. Используются для автоматического присвоения IP, если нет DHCP-сервера.
Применяются только внутри локальной сети, пакеты не маршрутизируются через роутеры, в интернет выйти не смогут. Только
L2,
L3 не работают.
