Подключение - это по сути применённый профиль настроек интерфейса и его маршрутизации.

`nmcli` - Network Manager, команда для отображения и управлениями сетевыми подключениями.
Изменения сохраняются в постоянную память в `/etc/NetworkManager/system-connections` и в `/etc/resolv.conf`
Может включать и выключать интерфейсы через команду `nmcli device`, как это делает `ip link`, но согласовано
с состоянием соединения. ip работает на уровне ядра.

`sudo nmcli connection modify "Spusk_5.0" ipv4.addresses 192.168.10.100/24` - назначение ip адреса подключения.
`sudo nmcli connection modify "Spusk_5.0" ipv4.gateway 192.168.10.1` - назначение шлюза
`sudo nmcli connection modify "Spusk_5.0" ipv4.dns 192.168.10.1` - назначение dns серверов
`sudo nmcli connection modify "Spusk_5.0" ipv4.method manual` - назначение статического метода присвоения ip адреса
интерфейсу
`sudo nmcli connection up "Spusk_5.0"` - активирует подключение

`auto` - ожидает, что получит адрес по dhcp
`manual` - статический адресация
`link-local` - автоматическое присвоение адреса в диапазоне подсети локальных адресов связи `169.254.0.0`
`disabled` - ipv4 адрес не будет назначен
`shared` - делает интерфейс NAT-хостом, раздаёт ip другим устройствам(wifi hotspot)

`sudo nmcli connection modify Spusk_5.0 +ipv4.routes "10.10.11.0/24 192.168.10.15"` - добавление постоянного
статического маршрута для соединения.

`sudo nmcli connection edit Spusk_5.0` - можно редактировать соединение в интерактивном режиме.
